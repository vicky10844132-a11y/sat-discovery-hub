# GS LinkOps AI — English-First Database Strategy

## 1. Core Decision

The database-building work for GS LinkOps AI should be English-first.

This does not mean Chinese materials are not useful.

It means the master database fields, source evidence records, technical terminology, platform interface and future partner-facing documents should primarily use English.

---

## 2. Why English First

Most global satellite, ground-station, GSaaS, standards, research and commercial materials are easier to collect, compare and reuse in English.

English is more suitable for:

```text
public satellite catalogues
GSaaS platform documentation
ground-station operator websites
API documentation
technical standards
research papers
conference papers
commercial capability sheets
international partner communication
investor-facing platform explanation
```

---

## 3. Database Benefits

An English-first database makes the system easier to scale globally.

Benefits include:

```text
consistent technical terminology
easier source extraction
easier field normalization
easier partner onboarding
easier comparison with AWS Ground Station, Leaf Space, KSAT, SSC, RBC Signals and other platforms
easier future API / documentation generation
easier investor and partner presentation
```

---

## 4. Recommended Language Structure

The recommended structure is:

```text
Master database: English
Platform interface: English first
Source evidence records: English
Technical field names: English
Partner-facing documents: English
Internal explanation notes: Chinese allowed
Owner decision notes: Chinese allowed
Commercial negotiation notes: bilingual when needed
```

---

## 5. Field Naming Rule

Database field names should use stable English technical terms.

Examples:

```text
satellite_name
norad_id
international_designator
operator
country_region
orbit_type
frequency_band
frequency_range
polarization
modulation
coding
bit_rate
frame_structure
antenna_size
rf_chain
baseband_system
booking_workflow
delivery_method
commercial_rule
risk_tags
confidence_level
source_evidence
missing_fields
next_action
```

---

## 6. Chinese Role

Chinese should be used where it helps the owner think, decide and explain.

Chinese is useful for:

```text
strategy discussion
business judgment
internal reasoning
contract negotiation with Chinese entities
translation of key concepts
owner decision records
```

But the operational database should stay English-first.

---

## 7. Product Rule

```text
Think in Chinese when needed.
Build the database in English.
Operate globally in English.
Translate only when necessary.
```

---

## 8. Product Conclusion

An English-first database will make GS LinkOps AI easier to build, easier to scale, easier to compare with global GSaaS platforms and easier to present to international partners.

This should become the default language strategy for the platform and database-building work.
