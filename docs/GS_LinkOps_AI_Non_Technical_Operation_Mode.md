# GS LinkOps AI — 非技术使用模式说明

适用对象：只有一个使用人员、没有开发人员、没有技术团队的情况下，由平台主人通过 ChatGPT 协助完成系统建设、维护、录入、检查和运营。

---

## 1. 核心原则

这个平台的建设方式不是：

```text
你自己写代码
你自己部署服务器
你自己研究数据库
你自己看 API 文档
```

而是：

```text
你负责提供业务信息和最终判断
ChatGPT 负责整理、设计、写代码、更新 GitHub、生成说明、检查逻辑、准备部署材料
平台负责把业务流程固定下来
```

你的角色是：

```text
平台拥有者 / 业务决策人 / 最终确认人
```

ChatGPT 的角色是：

```text
产品经理 + 技术方案师 + 开发执行助手 + 文档助手 + 运营助手
```

---

## 2. 你不需要懂什么

你不需要懂：

```text
GitHub
Python
FastAPI
PostgreSQL
Docker
Skyfield
SGP4
API
前端
后端
数据库
服务器部署
```

这些由 ChatGPT 持续帮你处理。

你只需要懂业务：

```text
有哪些卫星
有哪些地面站
谁是厂家
谁是合作方
任务怎么来
数据要给谁
谁确认
怎么计费
哪些信息敏感
哪些内容不能公开
```

---

## 3. 你需要提供的信息

每次要推进平台时，你只需要按业务语言给信息，不需要技术格式。

### 3.1 机构信息

你提供：

```text
公司名称
角色：卫星方 / 地面站 / 厂家 / 客户 / 供应商 / GS 内部
国家
联系人
邮箱
备注
```

ChatGPT 负责整理成平台字段。

---

### 3.2 地面站信息

你提供：

```text
地面站名称
国家/城市
经纬度
天线口径
支持频段
频率范围
极化
最低仰角
最大码率
解调器/接收设备
传输方式
是否已授权
成本价格
联系人
```

不知道的字段可以写：

```text
未知 / 待确认 / 后补
```

ChatGPT 会标记缺失项，不会乱填。

---

### 3.3 卫星信息

你提供：

```text
卫星名称
卫星编号
NORAD ID
TLE Line 1
TLE Line 2
频段
下行频率
极化
调制方式
编码方式
码率
帧格式
厂家/卫星方
是否授权
```

不知道也可以先空着，平台会通过 AI Copilot 提醒。

---

### 3.4 任务信息

你提供：

```text
谁要接收
接哪颗卫星
用哪个地面站或候选站
大概时间
任务目的：测试 / 正式接收 / 信号验证 / 商业任务
数据接收后给谁
怎么收费
```

ChatGPT 会帮助生成：

```text
Downlink Mission
Station Confirmation
Configuration Profile
Transfer Job
Billing Record
Mission Report
```

## 5. 你和 ChatGPT 的协作方式

推荐以后这样说：

### 添加资源

```text
我要把这个地面站加入平台，信息如下：……
```

### 检查信息

```text
帮我检查这个卫星信息能不能用于轨道计算和接收任务。
```

### 创建任务

```text
根据这个卫星和这个站，帮我生成一个测试接收任务。
```

### 生成报告

```text
根据这个任务状态，帮我生成给厂家看的接收报告。
```

### 计费判断

```text
这个任务接收到了信号，但是没有完整数据，应该怎么计费？
```

### 推进平台

```text
继续完善平台，把缺的模块补上。
```

## 6. 非技术使用流程

你日常真正需要操作的是这个流程：

```text
1. 把收到的卫星/地面站/厂家信息发给 ChatGPT
2. ChatGPT 整理成平台资源
3. ChatGPT 更新 GitHub 或生成导入文件
4. 你打开平台查看
5. 有任务时，你告诉 ChatGPT 任务需求
6. ChatGPT 帮你生成任务、报告、邮件、计费说明
7. 你确认后发给合作方或客户
```

---

## 7. 需要你最终确认的事情

这些必须由你决定，ChatGPT 不应替你擅自决定：

```text
是否加入某个合作方
是否认为某个站已授权
是否接受某个任务
是否给某个客户报价
是否确认计费
是否关闭任务
是否向外发送正式文件
是否共享敏感参数
```

---

## 8. ChatGPT 不应该做的事情

```text
不擅自加入未确认卫星
不擅自加入未确认地面站
不编造频率、TLE、码率、价格
不长期保存或公开敏感数据
不把测试任务当正式商业任务
不在未授权情况下生成真实接收任务
不替你发送正式邮件，除非你明确要求发送
```

---

## 9. 最适合你的平台形态

因为没有开发人员，平台应优先走这个路线：

```text
第一阶段：单页可运行平台 + GitHub 管理 + ChatGPT 协助
第二阶段：Docker 后端 + PostgreSQL + 你只看网页
第三阶段：部署到云端，ChatGPT 帮你维护和更新
第四阶段：资源、任务、报告、计费全部由平台和 ChatGPT 协同完成
```

你不需要管理代码本身。

你只需要说：

```text
这里不对
这里缺东西
这个功能要加
这个流程要改
这个报告要生成
这个任务要判断
```

---

## 10. 推荐的最终使用方式

最终平台应该让你做到：

```text
打开一个网页
看到所有资源和任务
点击或让 ChatGPT 帮你录入资源
自动算过境窗口
自动推荐地面站
自动生成接收任务
自动记录传输和厂家确认
自动判断能不能收费
自动生成报告和账单说明
AI 自动提醒哪里缺东西
```

你不需要知道背后代码怎么写。

## 11. 你给 ChatGPT 的标准指令模板

### 资源录入模板

```text
帮我把以下资源整理进 GS LinkOps AI 平台：
类型：地面站 / 卫星 / 厂家 / 客户
信息如下：
……
缺失字段你帮我列出来，不要乱填。
```

### 任务创建模板

```text
帮我根据以下信息创建一个下行接收任务：
卫星：
地面站：
时间：
任务性质：测试 / 正式
数据接收后给谁：
计费方式：
```

### 报告模板

```text
根据这个任务记录，帮我生成一份给厂家/客户看的报告。
语气正式，简洁，不暴露不该暴露的信息。
```

### 平台更新模板

```text
继续完善 GS LinkOps AI 平台，把这个功能加进去：……
```

你不用成为技术人员。
