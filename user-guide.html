1. Overview

The workbench is designed as a simple geospatial integration platform. It does not try to replace professional GIS software. Its purpose is to provide one operational entry point for AOI creation, user data reading, TLE pass prediction, KML/GeoJSON/CSV outputs, production task preparation, QC notes and owner-controlled delivery records.

2. Quick Start

  1. Open the homepage and choose the required module.
  2. Use Map Factory to draw AOI or manage project workflow.
  3. Use Tool Workbench to read user data or generate quick outputs.
  4. Use TLE Report to generate pass prediction report and vector files.
  5. Review Disclaimer & Sources before sharing outputs externally.

5. TLE Pass Report

The TLE Report page is used to generate professional satellite pass prediction outputs. It includes the TLE source, prediction settings, observer location, pass table, map coverage view and output files.

Required inputs

  • Satellite name;
  • TLE line 1 and line 2;
  • TLE source or provider;
  • Source URL or retrieval note;
  • Observer name, latitude and longitude;
  • Prediction window, step minutes and minimum elevation.

Outputs

  • Printable report / Save as PDF;
  • CSV pass table;
  • KML vectors for Google Earth;
  • GeoJSON vectors for GIS or web map;
  • Map coverage view.
Important: TLE-based prediction is for planning and preliminary assessment. It is not a certified precise orbit determination product.

8. Source Notes

Every report or output intended for external use should include source information. Missing source notes may create commercial or legal misunderstanding.

Recommended source note fields: - Dataset / TLE / Map layer name - Provider or source owner - Source URL or retrieval note - Retrieval date / generation date - Processing method - Output format - Limitation statement

9. Limitations

  • The preview system stores some data locally in browser storage and is not a final secure production backend.
  • Map drawings and coverage graphics are planning visuals, not guaranteed final production boundaries.
  • Open data and third-party layers remain subject to their original source terms.
  • User-uploaded data must be uploaded only when the user has the right to use it.
  • Commercial product scope, delivery, warranty and license must be governed by a project contract or quotation.

10. User Checklist Before Sharing Outputs

  • Check AOI boundary and coordinate system.
  • Check source information is included.
  • Check TLE source and retrieval date if using pass prediction.
  • Check map layer source and output file format.
  • Check disclaimer page has been reviewed.
  • Check whether the result is only preliminary or contract-grade.
