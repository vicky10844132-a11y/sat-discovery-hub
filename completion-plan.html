Required Improvements Before Final Launch

  1. Unify navigation: one clean entrance with Map, Tools, Products, Reports, Disclaimer.
  2. Reduce labels: avoid over-emphasizing open/self-owned labels in user interface; keep attribution inside output reports and disclaimer.
  3. Centralize source metadata: every report and export should include data source, retrieval note, map layer source, model/library and limitation.
  4. Real persistence: replace browser local storage with database such as Supabase, Firebase, Google Sheets or PostgreSQL.
  5. User permission: add admin/operator/client roles before accepting real external project data.
  6. Map engine decision: keep Leaflet for lightweight 2D functions; use CesiumJS for 3D globe if needed; use Google Maps or Google Earth Engine only when API and billing are ready.
  7. Report templates: standardize report cover, table of contents, map screenshot, pass table, source page and disclaimer footer.
  8. File workflow: support CSV, GeoJSON, KML, zipped vector packages and delivery manifest export.
  9. Security: avoid public exposure of sensitive AOI, client names, release codes or provider credentials.
  10. Deployment: deploy only after functions and disclaimers are stable, then point DNS to the final hosting platform.
