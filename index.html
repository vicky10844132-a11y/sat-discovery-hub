Platform Positioning

The Hub is a resource coordination and production orchestration platform. Storage, compute, data, algorithms, reference datasets, production partners and QC partners may be provided by different resource owners. The platform coordinates resources; it does not automatically own, expose or consume any provider's resources.

01 Resource Provider Registry

Register storage, compute, data, reference data, algorithm, production and QC providers.

02 AOI / Country Pack Request

Submit country, AOI, DSM/DOM specification, validation requirement and delivery format.

03 Production Factory

Match resources, split tasks, run mature production workflows and track status.

04 QC & Validation

Use public, commercial, client-provided or high-accuracy references for validation.

05 Owner-Only Tile Key Release

Generate tile-level delivery keys locked to owner release control. No direct credential delivery to third parties.

06 Delivery Center

Factory package with DSM, DOM, metadata, tile index, QC report, delivery note and license terms.

Factory Workflow

Data readiness check: coverage, stereo availability, cloud/snow/water risk.
Resource matching: storage, compute, algorithm and production partner.
Task split: AOI, grid and tile-level production tasks.
Basic production: DSM, DOM, orthorectification, mosaic and color balance.
QC intake and reference validation.
Factory release review and owner-only tile key generation.

Core Control Rule

All tile-level delivery keys and release credentials shall be issued exclusively to the platform owner's designated email address.

The system shall not send release credentials directly to clients, production partners, storage providers, compute providers or any third party. External release must be manually approved and forwarded by the platform owner.
